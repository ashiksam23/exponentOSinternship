# ExponentOS — Mentor Guide

A guide for mentors and facilitators running the 14-day ExponentOS internship program.

## Your Role

You're not a lecturer — you're a **guide**. Your job is to:

- Demo things live and narrate your thinking
- Get interns building as fast as possible
- Create a safe space where "I don't know" is celebrated
- Protect enough capstone time for every intern to finish something real
- Keep energy high, especially in a remote setting

## Before the Program Starts

- [ ] Test all tools yourself — create accounts, build the practice projects
- [ ] Set up the communication channel (Discord/Slack/Google Meet)
- [ ] Send welcome email with [Tools Setup Guide](tools-setup.md)
- [ ] Prepare the XP tracking spreadsheet (see [gamification/](../gamification/))
- [ ] Do a tech check call with each intern (camera, mic, internet)
- [ ] Flag that the 14-session calendar includes Monday, May 25, 2026 unless your organization observes that day as a holiday

## Running a Session

### Kickoff (10 min)
- Start on time, every time — consistency builds trust
- Ask a fun warm-up question tied to today's build
- Recap the previous session in 2 sentences
- State today's goal clearly: "By the end of today, you'll have ___"

### Learn Block (40 min)
- **Max 30% talking, 70% doing**
- Share your screen, build things live
- Make realistic mistakes and debug them out loud
- Pause every 5-7 minutes: "Try this on your own for 2 minutes"
- Use the chat for quick polls and check-ins

### Build Block (50 min)
- Start guided, gradually remove scaffolding
- First 15 min: follow along together
- Next 20 min: independent work with prompts/challenges
- Last 15 min: help those who are stuck, challenge those who are ahead
- **Never let someone sit stuck for more than 5 minutes** — check in proactively

### Play Block (40 min)
- This is NOT optional filler — it's the engagement engine
- Explain rules clearly before starting
- Keep time strictly — competitions need deadlines
- Award XP immediately — delayed rewards lose impact
- Celebrate effort, not just results

### Share & Wrap (20 min)
- Everyone shares something, even if it is a blocker
- Give specific shout-outs: "I loved how Priya's prompt used role-playing"
- Preview the next session: "Next time we're going to build ___"
- End 1 minute early — never run over

## Remote-Specific Tips

### Keeping Cameras On
- Don't force it, but encourage it
- Lead by example — keep yours on
- "I'd love to see your reactions when we demo this"

### Fighting Disengagement
- Call on people by name (rotate, don't pick favorites)
- Use breakout rooms for pair work (2 people max)
- Quick polls every 15 minutes ("Thumbs up if this makes sense")
- If someone goes quiet, DM them privately — don't call them out

### Screen Sharing
- Have interns share their screens during Build block
- Normalize messy work — "Let's debug this together"
- Use it as a teaching moment, not a performance review

## Handling Different Skill Levels

| Situation | Approach |
|-----------|----------|
| Someone finishes early | Give stretch challenges: "Can you make it do X too?" |
| Someone is stuck | Pair them with someone who finished, or debug together on screen |
| Someone is disengaged | DM them, ask what they'd rather build, adjust if possible |
| Someone is way ahead | Make them a "peer mentor" for the session — give them a badge for it |

## Giving Feedback

- **Be specific:** "Your prompt was strong because you gave the AI a role and an audience"
- **Be kind:** These are teenagers trying something new
- **Be constructive:** "What if you tried adding X?" not "This is wrong"
- **Celebrate the process:** "I love that you tried 5 different approaches"

## 14-Day Rhythm

| Days | Dates | Energy Pattern | Mentor Focus |
|------|-------|----------------|--------------|
| 1-4 | May 12-15 | Fast launch | Build confidence, normalize experimenting, introduce responsible AI |
| 5-7 | May 18-20 | Builder sprint | Push from AI user to AI product thinker |
| 8-10 | May 21-25 | Coding ramp | Keep code concrete, small, and immediately useful |
| 11-14 | May 26-29 | Ship mode | Cut scope early, prioritize a working capstone and portfolio |

## Capstone-First Facilitation Rules

- Treat Days 1-10 as skill-building, not a race to create polished portfolio pieces.
- On Day 11, every intern should choose a capstone that can be demoed in 2 minutes.
- By the end of Day 12, every capstone needs a working core flow, even if styling is rough.
- On Day 13, polish only what supports the final demo: reliability, clear inputs/outputs, README, screenshots, portfolio copy.
- On Day 14, celebrate shipped work. Do not let last-minute feature creep replace presentation practice.

## Emergency Toolkit

| Problem | Solution |
|---------|----------|
| Tool is down | Have a backup ready (ChatGPT <-> Gemini, Coze <-> Poe) |
| Intern can't join | Record the session, send catch-up materials |
| Dead silence after a question | Rephrase, give options, or answer it yourself and ask "Does that make sense?" |
| Technical meltdown | Switch to a discussion/brainstorm activity while fixing |
| Capstone is too ambitious | Help the intern keep one input, one AI call, one useful output |
| Someone is being unkind | Address privately, immediately — zero tolerance |
