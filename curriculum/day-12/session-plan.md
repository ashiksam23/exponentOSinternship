# Day 12 — Capstone Build

**Date:** Wednesday, May 27, 2026  
**Goal:** By the end of today, every intern will have a working capstone v1 with a complete core flow.

## Kickoff (0:00 - 0:10)

**Warm-up:** "What one feature must work today for your capstone to feel real?"

**Recap:** Yesterday we scoped capstones. Today we build the smallest working version.

**Today's goal:** "Your capstone should accept input, call AI, and show a useful output before the session ends."

## Learn Block (0:10 - 0:50)

### Concepts
- Build order:
  1. Inputs
  2. Prompt
  3. AI call
  4. Output
  5. Error handling
- Test with simple inputs before fancy ones
- Improve prompts after seeing real outputs
- Keep a bug log instead of panicking

### Live Demo
Add reliability to a simple AI script:

- Check for empty input
- Print a friendly loading message
- Catch API errors
- Ask the model for a predictable format
- Save a sample output for the README

### Pause Points
- "What is the smallest useful version?"
- "What error will users hit first?"
- "What output format makes the demo clear?"

## Build Block (1:00 - 1:50)

### Project: Capstone v1

Complete:

- User input flow
- Gemini API call
- Prompt with role, context, and output format
- Printed or displayed AI result
- One example run saved for portfolio/demo notes

**Stretch:** Add a menu, repeated loop, or second output option only after the core flow works.

## Play Block (2:00 - 2:40)

### Activity: Peer Build Room

- Can a peer run your project?
- Is the first prompt clear?
- Does the AI output match the goal?
- What is one confusing moment?
- What one fix matters most before tomorrow?
