# Day 13 — Capstone Polish + Portfolio

**Date:** Thursday, May 28, 2026  
**Goal:** By the end of today, interns will have a demo-ready capstone and a portfolio draft on GitHub Pages.

## Kickoff (0:00 - 0:10)

**Warm-up:** "What will make your demo easier for the audience to understand?"

**Recap:** Yesterday we got capstone v1 working. Today we polish what matters and package it for the world.

**Today's goal:** "Make your capstone reliable enough to present and build a portfolio page that explains it clearly."

## Learn Block (0:10 - 0:50)

### Concepts
- Polish is not adding random features
- Capstone polish priorities:
  1. Clear instructions
  2. Reliable input/output
  3. Better prompt
  4. Friendly errors
  5. Clean README
- Portfolio basics: headline, project cards, screenshots, links, reflection
- GitHub Pages publishing flow

### Live Demo
Use the portfolio template:

- Replace name and tagline
- Add capstone description
- Add supporting artifacts
- Add screenshots or links
- Publish with GitHub Pages

### Pause Points
- "Can someone understand the project in 10 seconds?"
- "What screenshot proves it works?"
- "What should be cut before final demo?"

## Build Block (1:00 - 1:50)

### Project: Portfolio Draft

Complete:

- Capstone title and description
- Problem/user statement
- Tools used
- Screenshot or sample output
- Link to Replit/GitHub project
- Supporting artifacts from earlier sessions
- Reflection paragraph

**Stretch:** Customize colors, layout, or badge section without breaking readability.

## Play Block (2:00 - 2:40)

### Activity: Portfolio Roast (Constructive)

Peers review portfolios for:

- Clear first impression
- Capstone is easy to find
- Project descriptions make sense
- Links work
- Text is readable
- One improvement before final demo

## Share & Wrap (2:40 - 3:00)

**Share prompt:** "Share your portfolio link or draft screen and name the one thing you'll fix before tomorrow."

**Next-session preview:** "Tomorrow is Final Demo and graduation. Practice your 2-minute story tonight."

## Post-Session Notes

- Award capstone demo-ready XP when the core flow works.
- Confirm every intern has a final demo order and presentation slot.
